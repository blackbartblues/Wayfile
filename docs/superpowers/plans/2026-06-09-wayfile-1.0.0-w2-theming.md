# Wayfile 1.0.0 — W2 Theming Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Realize the handoff's "one swappable accent" theming model — an obsidian-only base with 5 named accent presets, a live swatch picker in the Colours settings, and persistence — by deriving every accent-dependent token from a single `accent` value.

**Architecture:** The obsidian base, file-type tints, git colours and status colours are **constant** (they live in `ThemeLoader::s_defaults`). A preset TOML overrides **only** `accent` — so each preset is a one-line file. `Theme.qml` derives the gold ramp (`gold/goldMid/goldDeep/goldLight/goldInk`) and the alpha helpers (`goldGlow/goldWash/goldLine`) live from `accent` via an HSL shade helper, so swapping the accent retints all ~25 gold-ramp call sites + the gold-skinned Quill controls. The folder tint tracks the accent. Catppuccin is removed; the TOML-loading engine stays for `custom.toml`.

**Tech Stack:** Qt6/QML, C++ (ThemeLoader/ConfigManager), TOML themes, Qt6::Test (QCOMPARE/QSignalSpy). Build: `cmake --build build`; tests: `ctest --test-dir build`.

## Background / current state (read before starting)

- W1 (token adoption) is complete on branch `handoff-1.0.0`; `themes/bifrost.toml` carries the full obsidian+gold base, GUI-verified.
- `ThemeLoader` (`src/services/themeloader.{h,cpp}`) parses a `[colors]` TOML table into `m_colors`, falling back per-token to `s_defaults`. It exposes every token as a `Q_PROPERTY` + getter, plus `Q_INVOKABLE setColor/currentColor/saveThemeFile/colorKeys` for the granular editor. **`s_defaults` currently still holds the OLD Catppuccin/near-black palette** (e.g. `base #1e1e2e`, `gold #E3A94B`, `page #050609`) — W2 refreshes it to the obsidian base.
- `Theme.qml` (`src/qml/theme/Theme.qml`) is a singleton that mirrors each ThemeLoader token as a `property color X: theme.X`, and already derives `goldLine/goldGlow/goldWash` from `gold`.
- `FileTypeColors.qml` `folder` is a hardcoded literal `#D4AA6A`.
- The Colours settings page (`src/qml/components/SettingsSectionColors.qml`) is a granular per-token editor that writes `custom.toml` and switches `config.theme` to `"custom"`.
- The Look & Feel page (`src/qml/components/SettingsSectionLook.qml`) has a "Dark Mode" toggle (catppuccin light↔dark) and a "Theme" dropdown — both obsolete after W2.
- Theme selection persists via `config [general].theme`; `main.cpp` reloads `ThemeLoader` on `configChanged` (custom → `customThemePath()`, else `themesDir/<name>.toml`). **No new persistence plumbing is needed.**

## Locked W2 decisions (from brainstorm + this session)

- **5 presets:** Bifröst `#D4AA6A` (default) · Aurora `#57C7BF` · Nebula `#B292E8` · Ember `#E68B5C` · Verdant `#84C98A`.
- **Presets are minimal** (each TOML sets only `accent`); the obsidian base is inherited from `s_defaults`.

    // ── Accent-derived gold ramp (W2) ─────────────────────────────────────
    // A preset overrides only `accent`; the whole gold ramp + glow retints
    // from it. _shade shifts the accent's HSL lightness by `dl` and scales its
    // saturation by `ss` (hue preserved). The steps reproduce the W1 Bifröst
    // ramp for accent=#D4AA6A within ~±2/255.
    function _shade(c, dl, ss) {
        var h = c.hslHue >= 0 ? c.hslHue : 0
        var s = Math.max(0, Math.min(1, c.hslSaturation * ss))
        var l = Math.max(0, Math.min(1, c.hslLightness + dl))
        return Qt.hsla(h, s, l, c.a)
    }
