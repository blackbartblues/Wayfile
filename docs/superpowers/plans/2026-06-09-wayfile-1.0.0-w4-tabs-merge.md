# Wayfile 1.0.0 — W4 Tabs + Merge Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Adopt the handoff's simplified tab marker (a glowing gold top-bar shared by active + selected tabs — no border, no multi-select ring) and a binary chain-link merge button (armed gold when a merge/unmerge is available, dim + inert otherwise).

**Architecture:** `TabBar.qml` drops the active-tab border and the `selectedCount > 1` ring, and broadens the existing 2px gold top-bar marker (+ a soft accent wash) to `active || isSelected`. `Toolbar.qml`'s merge button becomes binary on a new `mergeOn` bool (from `Main.qml`); a new `IconMerge` (diagonal chain-link) replaces `IconLink`. Behavior (merge/unmerge/split) is unchanged except the button no longer fires when no merge/unmerge is available.

**Tech Stack:** Qt6/QML; `QtQuick.Shapes` (icon), `QtQuick.Effects` (`MultiEffect` glow). Build: `cmake --build build`; smoke: `ctest --test-dir build -R tst_qml_smoke`.

**Spec:** `docs/superpowers/specs/2026-06-08-wayfile-1.0.0-visual-system-design.md` §W4.

---

## Locked decisions

- **Tab marker = glowing gold top-bar** (the existing one), shown on `active || isSelected`. No border, no ring. `active` = obsidian gradient body + stronger accent wash (0.12) + white label; `selected`-not-active = gentler accent wash (0.07) + `subtext` label. Both share the top-bar.
- **No bloom-lift shadow / no ::before bridge** — they don't exist today, so nothing to remove (the spec lists them defensively).
- **Supertab art kept** (stacked tiles + ×N chip); only the outer ring is removed (it was a sibling, applied to all tabs).
- **Merge button binary:** `mergeOn = selectedCount >= 2 || activeTab.isSupertab`. ON → armed gold recipe + glow; OFF → opacity `.35` + inert. Icon = chain-link `IconMerge` for merge, `IconUnlink` for unmerge (supertab). **Trade-off (flag for user GUI review):** the old "click merge with no selection → merge-with-adjacent / split single tab" default is dropped — you now select ≥2 tabs to merge. Easy to restore if the user wants it.

## Task 1: (removed — chain-link glyph is INLINED in the merge button)

`src/qml/icons/` is a no-push git submodule (quill-icons); CLAUDE.md mandates **inlining new glyphs in the main repo (GitBadge.qml pattern)** rather than adding to the submodule. So there is **no new `IconMerge.qml` component** — the diagonal chain-link is inlined directly in the merge button (Task 3, Step 3) as a `Shape` with the two handoff `IcMerge` paths. The existing `IconUnlink` (already in the submodule) is reused for the unmerge glyph. No qmldir/CMakeLists/submodule changes.

## Task 3: Binary merge button

**Files:** Modify `src/qml/Main.qml` (add `mergeButtonOn()` + bind it), `src/qml/components/Toolbar.qml` (binary button + `IconMerge`).

- [ ] **Step 1: Add `mergeButtonOn()` in `Main.qml`**

Next to the existing `mergeButtonWillUnmerge()` function (~1025), add:
```qml
    // The merge button is binary (W4): armed only when a merge (≥2 tabs
    // selected) or an unmerge (active tab is a supertab) is available.
    function mergeButtonOn() {
        return !!(tabModel.activeTab
                  && (tabModel.activeTab.isSupertab || tabModel.selectedCount >= 2))
    }
```

- [ ] **Step 2: Add a `mergeOn` property to the Toolbar instance**

Where `Main.qml` instantiates the `Toolbar` and sets `mergeWillUnmerge`/`mergeTooltip` (~1610), add the binding:
```qml
                mergeOn: root.mergeButtonOn()
```
(alongside the existing `mergeWillUnmerge: root.mergeButtonWillUnmerge()` and `mergeTooltip: root.mergeButtonTooltip()`).

- [ ] **Step 3: Declare `mergeOn` + rewrite the merge button in `Toolbar.qml`**

At the top of `Toolbar.qml` near `property bool mergeWillUnmerge: false` (~21), add:
```qml
    property bool mergeOn: false
```

Replace the merge button `HoverRect` (~183-213) with the binary version:
```qml
    HoverRect {
        id: mergeBtn
        width: Theme.controlSize; height: Theme.controlSize
        visible: !root.searchMode
        enabled: root.mergeOn                 // inert when no merge/unmerge available
        opacity: root.mergeOn ? 1.0 : 0.35    // dim when off (binary, no neutral state)
        border.width: 1
        border.color: Theme.gold              // 1px accent border (armed recipe)
        gradient: mergeArmedGrad
        color: "transparent"
        layer.enabled: root.mergeOn
        layer.effect: MultiEffect {
            autoPaddingEnabled: true
            shadowEnabled: true
            shadowColor: Theme.goldGlow
            shadowBlur: 0.7

        onClicked: root.splitViewToggled()

        Gradient {
            id: mergeArmedGrad
            GradientStop { position: 0.0; color: Theme.goldLight }
            GradientStop { position: 0.6; color: Theme.gold }
            GradientStop { position: 1.0; color: Theme.goldMid }

            }
            ShapePath {
                strokeColor: Theme.goldInk; strokeWidth: Math.max(1, 18 / 12)
                fillColor: "transparent"; capStyle: ShapePath.RoundCap; joinStyle: ShapePath.RoundJoin
                scale: Qt.size(18 / 24, 18 / 24)
                PathSvg { path: "M13.6 10.4a4 4 0 0 0-6-.43l-2.2 2.2a4 4 0 0 0 5.66 5.66l1.25-1.25" }
            }
        }
        IconUnlink { anchors.centerIn: parent; size: 18; color: Theme.goldInk; visible:  root.mergeWillUnmerge }
        Q.Tooltip { text: root.mergeTooltip; visible: mergeBtn.hovered && root.mergeTooltip.length > 0 }
```
