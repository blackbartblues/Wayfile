# Wayfile 1.0.0 — Visual System Design Spec

**Date:** 2026-06-08
**Status:** Design approved by user (brainstorm). Ready to spawn per-workstream implementation plans.
**Branch:** new branch off `main` (local `main` @ the post-gallery state).

> **Scope-now vs later (user, 2026-06-08):** the **1.0.0 push/release is deferred to later** —
> the version bump, screenshot re-shoot, and all publish mechanics (push `main`, repo
> rename/public, tag, AUR, Flathub) are **not** part of the current effort. **Now** we
> implement the handoff's *visual* workstreams **W1–W6** only. The "Release" section below is
> documented for completeness but stays parked until the user calls for it.

## Goal

Implement the new hi-fi design handoff as Wayfile's **"real" 1.0.0** — the first public
release. The handoff is a complete, code-accurate visual system (obsidian base, a single
swappable accent, a thin-frame glowing icon language, 5 themes, a compass-folder logo).
Much of the app's chrome (sidebar rail, breadcrumb pills, pane strips, list columns,
context menu, status bar) already matches the handoff from earlier work and is treated as
*validation*; this effort is the set of genuine deltas.

This is a **large, multi-subsystem effort** → decomposed into six workstreams (W1–W6),
each getting its own implementation plan (`writing-plans`) and its own
implement → review cycle (subagent-driven-development). This document is the single
overarching design that all six plans derive from.

## Handoff source (on-disk reference, gitignored)

`docs/design-handoff/design_handoff_wayfile/` — extracted from `~/Downloads/Heimdall(4).zip`
(2026-06-08, the finalized version). It is **gitignored** (like the prior handoffs) and
serves as the implementation reference. Key files:

- `README.md` — prose truth (component specs, icon system, interaction/behaviour).
- `tokens.css` — **source of truth for values** (every CSS var + every component class + the 5 `.theme-*` blocks).
- `wayfile-icons.jsx` — `WayFolder` / `WayFile` / `FileIcon` dispatcher, `IcMerge`, `WayfileLogo`, all glyph SVG paths + the per-state glow opacity tables.
- `wayfile-components.jsx`, `wayfile-artboards.jsx`, `wayfile-board.jsx` — composition / interaction (React-via-Babel prototype; structure, not ship code).

The handoff prefix is `--h-` / `.h-` (legacy "Heimdall" codename). We port values into
Wayfile's existing token layer; we do **not** keep the `--h-` prefix.

## Locked decisions (from brainstorm)

1. **Tokens (W1):** adopt `tokens.css` values **1:1** — the handoff's *lighter, steel-grey
   obsidian ladder* and *softer tan-gold* `#D4AA6A` replace the current near-black base
   (`#111217`) and warmer gold (`#E3A94B`). The handoff is the source of truth for values.
2. **Theming (W2):** **obsidian-only + 5 accent presets.** The obsidian base never moves;
   a theme re-points only the accent (and the folder tint, which tracks it). File-type tints
   and git colours never move. Light/catppuccin themes are **not shipped as blessed** (the
   TOML engine stays; the glow-icon language is dark-only). Live switch + persisted.
3. **Icons (W3):** adopt the handoff's **thin-frame icon ART** (folder frame; file =
   page-frame + dog-ear + per-type line-motif, **no chips**). **Resting frame colour:** files
   in their **type colour** (pdf red, `.ts` teal, …), folders **gold/accent**. **Highlight on
   hover/select = the user's existing gold bloom** (`MultiEffect`, `Theme.goldGlow`) — a
   *uniform gold* halo, **not** the handoff's type-coloured glow — **extended to every view**
   (today it exists only in the grid). `FileTypeChip` is retired; the dormant glossy
   `OsFolder`/`OsFile` are superseded.
4. **Tabs + merge (W4):** tab selection marker = a **glowing top bar only**, no border/ring,
   no lift-shadow; `active` and merge-`selected` share it; merged tab = bar + stacked-icon +
   `×N` chip. **Merge button is binary** — ON (armed gold glow) / OFF (disabled .35), no
   neutral resting state; merge icon = diagonal **chain-link**.
5. **Logo (W5):** new **compass-folder `WayfileLogo`** (variants `app` / `gold` / `mono`;
   the compass accent retints with the theme) replaces the stone gateway-folder, including
   app/window/launcher icons. **No wordmark** — the tab strip stays flush-left.
6. **Typography (W6):** **Inter** (UI) + **JetBrains Mono** (mono) + **Cormorant Garamond**
   (display: theme names / large labels). Cinzel is removed.
7. **Version:** this release is **1.0.0** (the local `1.0.1` was never published; the public
   debut is the "real 1.0.0"). The whole handoff is the content of 1.0.0.

**Cross-cutting invariants:** `FileTypeColors` and `GitColors` are **fixed** across all
themes. Per-icon glow is applied **only on hover/selected** (rest = plain `Shape`, no
effect), matching the existing grid mechanism and the `GitBadge` faux-shadow lesson (no
per-element blur on hundreds of resting rows).

---

## Build order (dependencies)

**W1 tokens → W2 themes → W3 icons → W4 tabs/merge → W5 logo → W6 typography → release.**

- Tokens first: everything paints on them.
- Themes second: the icon glow + selection read the accent, so the accent-derivation must exist before icons.
- Icons third: the biggest workstream; depends on the (new) `FileTypeColors` values + accent.
- Tabs/merge, logo, typography are largely independent cosmetics; ordered last.
- Release (version bump, screenshots) after all six.

Each workstream is independently buildable, testable, committable, and shippable behind the
others.

## W1 — Token adoption (foundation)

**Scope:** Re-point Wayfile's token layer to the handoff's exact values. Values only — no
structural/component changes.

**What changes (handoff → our layer):**

- **Obsidian ladder:** stone `#2B2F33` (window/canvas) · surface `#1E2126` (panels) ·
  surface-alt `#25292E` (raised: toolbars, idle tabs) · surface-hi `#2F343A` (hover/active
  rows) · border `#353B42` · border-soft `#2A2E33` · divider `#1A1D21`. Darkest insets
  (search/breadcrumb wells) `#15181C`.
- **Text:** primary `#E6E1D6` · secondary `#A3A8AE` · tertiary/disabled `#6C7177`.
- **Accent (default Bifröst gold):** `#D4AA6A` (rgb `212,170,106`) · soft `#B8915A` · folder
  tint `#D4AA6A` (tracks accent). `accent-glow = rgba(accent,0.18)`, `accent-line = rgba(accent,0.45)`.
- **File-type tints (FIXED):** image `#8FC380` · doc `#6FA8DC` · md `#BFA4E0` · pdf `#E06C75`
  · zip `#E0B26C` · code `#7BC6BD` · audio `#D4A6A6` · video `#C68FE0` · txt/cfg/bin →
  text-2/text-3. → port into `FileTypeColors.qml`.
- **Status:** success `#56B881` · danger `#E06C75` · info `#6FA8DC`.
- **Radii:** sm 4 · md 6 · lg 10 · **xl 14** (window).
- **Shadows / metal:** `shadow-sm/md/lg` + metallic sheen
  `linear-gradient(180deg, rgba(255,240,215,0.05) 0%, …0 30%, rgba(0,0,0,0.10) 100%)` + inset
  rim `inset 0 1px 0 rgba(255,240,215,0.06), inset 0 -1px 0 rgba(0,0,0,0.4)`. Window catch-light
  `radial-gradient(120% 60% at 50% 0%, rgba(255,240,215,0.04), transparent 60%)` + vertical
  metal gradient `#2D3137 → #25292E → #1E2126`.

**Reconciliation note (the real work):** the current `themes/bifrost.toml` carries a richer
token set than the handoff lists — semantic aliases (`base/mantle/surface/overlay/text/
subtext/accent`), a gold ramp (`gold/goldMid/goldDeep/goldLight`), `panel/panel2`, atmosphere
(`sheen/shadowInk`), and derived `goldGlow/goldWash/goldInk`. These must be **re-derived** from
the new lighter base + new gold so the obsidian-and-gold feel survives the re-tint. Map the
handoff's `--h-*` onto the existing semantic names (no rename churn in components).

**Files:** `themes/bifrost.toml` (primary), `src/qml/theme/Theme.qml` (any non-token derivations;
see W2 for the accent-derivation refactor), `src/qml/theme/FileTypeColors.qml` (adopt the
fixed file-type tints), atmosphere tokens. `GitColors.qml` unchanged.

**Out of scope:** component structure; theming model (W2); any icon/tab/logo/font work.

**Testing:** `cmake --build build` + `tst_qml_smoke` + `ctest`. GUI: the whole app re-tints
to the lighter obsidian + softer gold with no broken/again-too-dark surfaces.

## W2 — Theming: obsidian-only, 5 accent presets

**Scope:** Realize the handoff's "one swappable accent object" model on the fixed obsidian
base, shipping 5 named themes, with a live picker + persistence.

**Model:** A theme overrides only the accent. To truly make "swap one accent → retint
everything" real (and to keep presets to one-line overrides), **derive the accent-dependent
tokens from the single accent** in `Theme.qml`:

- `gold == accent`; `goldGlow = rgba(accent, 0.30)`; `goldWash = rgba(accent, 0.08)`;
  `accentLine = rgba(accent, 0.45)`; the gold ramp (`goldMid/goldDeep/goldLight`) and
  `goldInk` derived from the accent by fixed lightness/saturation steps.
- The obsidian base, file-type tints, git colours, and status colours are **constant** across
  presets.

**5 presets** (TOML files sharing the identical obsidian base; each sets only `accent`,
`accent-soft`, `ft-folder`):

| Preset | Accent | Note |
|---|---|---|
| Bifröst (default) | `#D4AA6A` | warm gold |
| Aurora | `#57C7BF` | cool aqua-teal |
| Nebula | `#B292E8` | soft violet |
| Ember | `#E68B5C` | warm coral |
| Verdant | `#84C98A` | sage green |

**Picker + persistence:** surface the 5 presets in the existing **"Colours"** Settings
section (the Phase-C surface) as selectable swatches; `ThemeLoader.loadTheme` applies live;
persist the chosen preset (ConfigManager theme key). The existing token editor may remain for
power users but is de-emphasized; the headline is the 5-swatch picker. Light/catppuccin TOMLs
are dropped from the blessed/shipped set (engine kept; they may live as unsupported extras).

**Files:** `themes/{bifrost,aurora,nebula,ember,verdant}.toml`; `src/qml/theme/Theme.qml`
(accent-derivation); `src/services/themeloader.{h,cpp}` (if derivation moves to C++); the
Colours Settings component; `src/services/configmanager.*` (persist active preset);
remove catppuccin from the shipped themes list.

**Out of scope:** light mode; per-theme file-type/git colours; a full theme-authoring UI.

**Testing:** unit-test the accent-derivation (a given accent → expected glow/wash/ramp) if it
lands in C++; switch all 5 presets live; verify persistence across restart; verify file-type
+ git colours are identical across presets. Smoke + GUI verify each preset.

## W3 — Icon system (thin-frame art + gold-bloom highlight) — biggest workstream

**Scope:** Replace the current flat-glyph + `FileTypeChip` file/folder rendering with the
handoff's thin-frame icon language, and extend the existing gold-bloom hover/select highlight
to every view.

**New icon components (QML `Shape`, mirroring the handoff `WayFolder`/`WayFile`/`FileIcon`):**

- **`WayFolder.qml`** — thin folder-silhouette frame (handoff path), stroked in the **folder
  colour = accent/gold**, round joins/caps; an optional subtle interior radial bloom; optional
  `emblem` slot for typed folders (Home/Downloads/Documents, kept from current typed-folder
  work).
- **`WayFile.qml`** — thin page frame + dog-ear, stroked in the file's **type colour**
  (`FileTypeColors`), with a single-weight **line-motif** per kind inside (md `M↓`, code `< >`,
  image sun+horizon, audio note+beams, video play △, zip dashed seam, cfg gear, bin bars,
  doc/txt/pdf three lines). **No text chip.**
- **`FileIcon.qml`** dispatcher — `kind` → `WayFolder` (accent) or `WayFile` (type colour +
  motif). Replaces both `IconFolder`-as-folder and `FileTypeChip`-as-file at call sites.

**Highlight (state-driven, the user's existing language):** on **hover/selected**, wrap the
icon in the existing `MultiEffect { shadowColor: Theme.goldGlow; shadowBlur: 0.45 hover / 0.7
selected }` — a **uniform gold bloom**, *not* type-coloured. At **rest there is no effect**
(just the `Shape`). Factor this into a small reusable wrapper and apply it in **every** view,
since today only the grid has it.

**Per-view application:**

| View | File | Today | After |
|---|---|---|---|
| Grid | `FileGridView.qml` | gold bloom (line ~638) on `IconFolder`+`FileTypeChip` | same bloom on `FileIcon` (WayFolder/WayFile) |
| Detailed rows | `FileDetailedRow.qml`/`FileDetailedView.qml` | no per-icon glow | `FileIcon` + gold bloom on hover/select |
| Miller | `FileMillerView.qml`/`MillerCurrentRow.qml`/`MillerPreviewColumn.qml` | no per-icon glow | `FileIcon` + gold bloom on hover/select |
| Gallery filmstrip | `GalleryView.qml` (reuses grid) | inherits grid bloom | inherits (FileIcon) |
| Gallery folder tree | `GalleryFolderNav.qml` | gold `IconFolder`, no glow | `WayFolder` + gold bloom on hover/current |
| Sidebar | `Sidebar.qml` | place glyphs, glow on left-bar only | keep places as-is; folder/bookmark rows may adopt `WayFolder` (plan decision) |

**Performance:** the gold bloom is applied **only to hovered + selected** icons (rest = plain
`Shape`), so at most a handful glow at once across a list of hundreds — the proven pattern.
Thumbnails (image/video) still win over the icon, unchanged. The merged-selection outline
(Canvas in grid, run overlays in detailed/miller) stays as-is.

**Retire:** `FileTypeChip.qml`; `OsFolder.qml`/`OsFile.qml` (dormant glossy art, superseded —
delete or leave unreferenced per plan).

**Files:** new `src/qml/views/{WayFolder,WayFile,FileIcon}.qml` (+ register in
`src/CMakeLists.txt` `QML_FILES`); modify the six rendering views above; the new line-motif
glyphs; `FileTypeColors` already updated in W1.

**Out of scope:** selection/merged-outline logic; thumbnail pipeline; the handoff's
type-coloured *glow* (we use uniform gold); interior-bloom/rim-lip fidelity beyond a subtle
touch (finalize in the W3 plan).

**Testing:** smoke; per-view GUI verify (rest = type-coloured file frames + gold folder
frames; hover/select = gold bloom in **every** view); large-directory perf check.

## W4 — Tabs + merge

**Scope:** Adopt the handoff's simplified tab marker and binary merge control.

- **Tabs (`TabBar.qml`):** selection marker = a **2px accent top bar** (`transparent → accent
  22–78% → transparent`, glow `0 0 7px α.80, 0 0 16px α.45`) + a soft top-down accent wash
  (`rgba(accent,0.12) → 0 @44%`) + `inset 0 1px 0 rgba(255,240,215,0.05)`. **Remove** the
  border/ring, the `-6px 18px` bloom-lift shadow, and the `::before` toolbar bridge. `active`
  and merge-`selected` share the bar; selected-not-focused = gentler wash + `text-2` label.
  Merged tab = the same bar + the stacked-icon cluster + `×N` chip (keep existing supertab
  art, restyled to no-ring).
- **Merge (`Toolbar.qml`):** binary — a single `mergeOn` bool. ON → `.h-iconbtn--armed`
  recipe (radial gold fill, 1px solid accent border, outer+inset glow, glyph drop-shadow).
  OFF → disabled (opacity .35), inert. **No** neutral/idle-gold-wash state. Icon = diagonal
  **chain-link** (handoff `IcMerge` path), 1.5px round-cap, on the shared icon wrapper.

**Files:** `TabBar.qml`, `Toolbar.qml`, a new chain-link merge icon, and the `mergeOn`
plumbing (Main.qml exposes "merge available" as a single bool).

**Out of scope:** the merge→split behaviour itself (already built); tab DnD.

**Testing:** smoke; GUI verify a selected tab is never boxed, merge button is ON only when a
merge is available and OFF/dim otherwise.

## W5 — Logo (compass-folder), no wordmark

**Scope:** Build the new vector logo and swap it in for the app/window/launcher icons and the
chrome mark. No wordmark.

- **`WayfileLogo.qml`** (QML `Shape`): a folder cradling a faceted gold compass star (4-point
  star = 8 alternating light/dark blades, thin steel ring + 8 ticks + hub). `variant`:
  - `app` — glossy dark folder (`#343A41 → #262A30 → #1A1D22`, 1px top highlight) + gold
    compass; `plate` wraps it in a 26px-radius squircle (`#2B3037 → #15181C`, top sheen).
  - `gold` — symbolic gold line/fill (toolbars/splash/About).
  - `mono` — neutral steel `#9CA0A8` (trays/disabled).
  - The compass needle/accent **retints with the active theme**; the obsidian folder shell is
    fixed. Legible 16 → 140px. `WayfileMark` alias = `app` variant for chrome.
- **Replace:** the chrome leading mark (tab strip / sidebar header) with `WayfileMark` (icon
  only, no wordmark); the window icon (`main.cpp setWindowIcon`); the app/launcher icon
  (`dist/io.github.blackbartblues.Wayfile.svg`, regenerated from the vector); `src/assets/`
  logo bitmap if still referenced.

**Files:** new `src/qml/.../WayfileLogo.qml` (+ QML_FILES); `TabBar.qml`/`Sidebar.qml` header;
`src/main.cpp`; `dist/*.svg`; `src/assets/`.

**Out of scope:** any "Way·file" wordmark text.

**Testing:** GUI verify the mark renders cleanly 16–140px and retints per theme; window +
launcher icon resolve.

## W6 — Typography

**Scope:** Adopt the handoff font stack.

- Bundle **Inter** (UI) and **Cormorant Garamond** (display); keep **JetBrains Mono** (mono);
  remove **Cinzel** (dormant) and its references.
- Set the UI font family to Inter (enable `cv11, ss01, ss03`, base 13px / 1.45), display to
  Cormorant (theme names / large labels), mono to JetBrains Mono (sizes, counts, paths, type
  chips; tabular-nums).

**Files:** `src/qml/theme/Fonts.qml` (or wherever font families are defined) + `Theme.qml`
font tokens; bundled font assets + qrc registration; remove Cinzel asset/refs.

**Out of scope:** per-widget type-scale redesign beyond the handoff's named sizes.

**Testing:** smoke; GUI verify body text is Inter, theme names are Cormorant, mono unchanged.

## Testing strategy (all workstreams)

- Each workstream: `cmake --build build` → `tst_qml_smoke` → `ctest --test-dir build -j1` →
  user GUI-verify on Wayland (QML is rcc'd into the binary — always rebuild before smoke).
- Add Qt6::Test unit tests where C++ logic is introduced (e.g., accent-derivation in
  `ThemeLoader`).
- Reuse the existing `tst_qml_smoke` safety net (loads the whole tree headless, catches
  QML load/binding/required-property errors).
- Final: full suite green + refreshed screenshots before the version bump.

## Risks / open items (resolve in the per-workstream plans)

- **Token reconciliation (W1/W2):** deriving the gold ramp / atmosphere / glow from a *single*
  accent must look good for all 5 hues (teal/violet/coral/green), not just gold — verify the
  derived `goldGlow/goldWash/ramp` per preset.
- **Icon perf at scale (W3):** validated approach (glow only on hover/selected); confirm on a
  large directory; watch `autoPaddingEnabled` buffers.
- **Interior bloom vs external gold bloom (W3):** the handoff art carries an interior bloom +
  rim lip-light; we use the external gold `MultiEffect` as the highlight — decide in the W3
  plan how much interior glow (if any) to keep at rest.
- **Logo as packaging asset (W5):** the vector must export cleanly to the `dist/*.svg` app icon
  and resolve as window/launcher icon (Wayland app_id already set).
- **Versioning (release):** `dist/*.metainfo.xml` currently has both 1.0.0 and 1.0.1 entries —
  reconcile to one 1.0.0 public release.

## Out of scope (global)

- Light mode / catppuccin as blessed themes (engine retained; not shipped).
- A full theme-authoring UI beyond the 5-preset picker + accent tweak.
- Changes to selection/merged-outline logic, the merge→split behaviour, thumbnails, or the
  preview/video/metadata engines.
- The publish/release mechanics (push/rename/tag/AUR/Flathub) — they follow implementation.
